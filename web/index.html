⚡ 快速开始

1. 启动代理服务

cd ~/Claude/proxy-rewrite ./proxy-rewrite

2. 配置浏览器或系统代理为 127.0.0.1:8080

3. 在「转发规则」页面添加你需要的规则

4. 通过代理的请求会自动匹配并重写转发

🌐 浏览器代理配置

Firefox (推荐) 设置 → 常规 → 网络设置 → 手动代理配置：

HTTP 代理127.0.0.1端口 8080

Chrome 可使用 SwitchyOmega 扩展，配置 HTTP 代理 127.0.0.1:8080

curl 测试：

curl -x http://localhost:8080 http://api.example.com:8080/v1/test # 会被规则匹配并重写转发到目标地址
💡 通配符规则说明

在匹配模式和目标地址中使用 * 作为通配符，支持多个 *：

匹配模式目标地址说明
http://api.example.com:8080/v1/*http://127.0.0.1:3000/v1/*替换主机和端口，路径保持不变
http://*.example.com:*/api/*http://127.0.0.1:3000/api/*通配符匹配任意子域名和端口
https://*/api/v1/*http://localhost:8080/v1/*任意域名，HTTPS 转 HTTP
http://*:*/mock/*http://127.0.0.1:9999/*任意主机任意端口的 mock 路径

规则匹配成功后，* 捕获的内容会按顺序填入目标地址中对应的 * 位置。当匹配模式的 * 比目标多时，从右往左对齐（路径优先）。

未匹配任何规则的请求会直接透传到原始地址。

📄 配置文件

规则存储在 data/rules.json，格式：

{ "proxy_port": "8080", "web_port": "9090", "rules": [ { "name": "my-api", "match_pattern": "http://api.example.com:8080/v1/*", "target_url": "http://127.0.0.1:3000/api/*", "enabled": true } ] }

可通过 Web 页面管理，也可直接编辑此文件后重启服务生效。

新增规则

支持 * 通配符，例如 http://*.example.com:*/api/*
匹配模式中的 * 会按顺序填入此处对应的 *
